# Contributing to VHAKACHAYI Cultural Tours

Thank you for your interest in contributing to VHAKACHAYI Cultural Tours. This project operates under a **Cultural Sensitivity Override**, meaning all contributions must honor tradition first.

## Cultural Review Process

**CRITICAL**: All changes must be reviewed by our Elder Advisory Board before merging. This includes:

- Content changes
- Design modifications
- Feature additions
- Translation updates

## Development Guidelines

### 1. Cultural Respect

- Always prioritize cultural protocols over technical requirements
- Consult cultural guidelines before making changes
- Respect sacred site terminology and protocols
- Ensure all content honors tradition

### 2. Code Standards

- Follow TypeScript best practices
- Use Tailwind CSS for styling
- Maintain accessibility standards
- Write clear, documented code

### 3. Translation Guidelines

- Shona and Ndebele translations must be reviewed by native speaker elders
- Cultural nuances and proverbs must be accurate
- Respect oral tradition in written content

### 4. Testing

- Test multilingual functionality
- Verify cultural protocol features
- Check accessibility (screen readers, etc.)
- Test payment flows thoroughly

## Pull Request Process

1. **Create a feature branch** from `main`
2. **Make your changes** following guidelines
3. **Submit for cultural review** (required)
4. **Address feedback** from both technical and cultural reviewers
5. **Merge** after approval

## Areas Needing Contribution

- Additional Shona/Ndebele translations
- Cultural calendar dates
- Elder interview content
- Community impact stories
- Accessibility improvements

## Questions?

Contact the project maintainers or consult the Elder Advisory Board for cultural guidance.

**Remember**: The old ways carry the deepest magic. Honor tradition in all contributions.
